# Шаблон фич — соответствие ТЗ (текстовая копия для сдачи)

Заполните одноимённый файл **GPTHub шаблон фич.xlsx** теми же формулировками или скопируйте колонки отсюда.

| № | Функционал | Сделано | Как и через что |
|---|------------|---------|-------------------|
| 1 | Текстовый чат | Да | `POST /v1/chat/completions` через шлюз → MWS GPT; модель из роутера (`gpthub-auto`) или вручную. |
| 2 | Голосовой чат | Да | Аудио в сообщении → `POST /v1/audio/transcriptions` (Whisper в MWS) → текст в тот же тред → LLM. |
| 3 | Генерация изображений в чате | Да | Ключевые фразы / intent → `POST /v1/images/generations` (например `qwen-image` из каталога). |
| 4 | Аудиофайлы + авто ASR | Да | Вложение аудио → транскрипция Whisper → дальше обычный чат. |
| 5 | Изображения (VLM) | Да | `image_url` / картинка в сообщении → vision-модель из env (`VISION_MODEL`), подбор из `/v1/models`. |
| 6 | Файлы и ответы по содержимому | Да | RAG: чанки, `POST /v1/embeddings` с `bge-m3`, SQLite в volume → контекст в chat completions. |
| 7 | Поиск в интернете | Да | DuckDuckGo (`duckduckgo-search`) → фрагменты в system → ответ только LLM через MWS. |
| 8 | Веб-парсинг по ссылке | Да | trafilatura по URL из текста → фрагмент в контекст → LLM. |
| 9 | Долгосрочная память | Да | SQLite + эмбеддинги bge-m3, retrieval в system; история треда не затирается. |
| 10 | Автовыбор модели | Да | `gpthub-auto`; правила **gena** (`pick_route_gena`, перехваты презентация/картинка/deep research); режим `legacy` — `pick_route_deterministic`; префикс `provider/model` нормализуется. |
| 11 | Ручной выбор модели | Да | Любой id из `/v1/models` кроме авто — шлюз не подменяет `model`. |
| 12 | Markdown и код | Да | Рендер Open WebUI. |
| 13 | Deep Research (доп.) | Частично | Несколько запросов DDG + контекст для одного ответа LLM (`deep_research_ddg`, фразы «глубокое исследование» / `deep research`). |
| 14 | Презентации (доп.) | Нет / вручную | Не автоматизировано; можно экспортировать ответы LLM в PPTX вне репо. |
| 15 | Прочий бонус | — | По необходимости. |
