# ТЗ GPTHub — приёмка по тестам

**Правило:** статус **«готово»** и галочки в чекбоксах ставьте **только после** успешного прохождения указанных тестов. До этого всё остаётся незакрытым.

**Как отмечать:** замените `[ ]` на `[x]` у пройденного теста; в строке **«Статус пункта ТЗ»** поставьте ✅ и дату только когда **все** тесты этого пункта закрыты. Кратко фиксируйте результат (ОК / не ОК) в поле «Заметки».

**Окружение перед любыми тестами:** `docker compose ps` — шлюз **healthy**; в `.env` актуальный **`MWS_API_KEY`**. В UI по умолчанию модель **`gpthub-auto`**, если в тесте не сказано иное. Подробности: [QA_CHECKLIST.md](QA_CHECKLIST.md), [FEATURES_CHECKLIST.md](FEATURES_CHECKLIST.md).
